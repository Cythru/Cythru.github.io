About

Who we are

Cythru is an open-source engineering collective focused on low-level systems software. We write close to the metal — operating system internals, network stacks, compilers, and security tooling — prioritizing correctness, performance, and long-term maintainability over trend-driven complexity.

Our work is public, our decisions are transparent, and our output belongs to the community. We believe the most important software in the world should be understandable by anyone willing to read it.

Philosophy

How we build

Modular by default

Every component is independently buildable, testable, and replaceable. Monolithic coupling is a design failure, not a shortcut.

No unnecessary abstractions

Abstractions are introduced only when they remove real complexity — not to satisfy patterns or conventions. Readability beats cleverness.

Own your dependencies

We audit what we pull in. Shallow dependency trees are a feature. If we can write it small and correct ourselves, we do.

Measure everything

Performance claims require benchmarks. Security claims require proofs. Intuition is a starting point, not a conclusion.

Stack

Languages we trust

C++

The lingua franca of systems work. Decades of tooling, precise control over memory layout, and zero-overhead abstractions when you need them.

Rust

Memory safety without a garbage collector. Ownership semantics enforce correctness at compile time — the right trade-off for security-critical components.

Zig

Comptime evaluation, no hidden control flow, and a build system that actually works. Our choice when we need C-level simplicity with modern ergonomics.

Initiative

VoidForge is Cythru's open-source commitment — everything we build is released under free software licenses and developed in the open from day one. No delayed source drops, no open-core bait-and-switch, no proprietary forks.

We forge systems from nothing: no inherited cruft, no legacy obligations, no design-by-committee. Each project starts with a blank slate and a clear specification.
